# ФАБРИКА интерьеров — Сайт-каталог

Современный сайт-каталог дизайнерской мебели и интерьерных решений с функцией презентации бренда и сбора заявок.

## 📋 Описание проекта

Сайт создаётся как витрина продукции компании "ФАБРИКА интерьеров" — производство полного цикла интерьерных решений. Основная функция — демонстрация продукции и лидогенерация (сбор заявок от потенциальных клиентов).

**Целевая аудитория:**
- Дизайнеры интерьера
- Девелоперы
- Частные клиенты

## 🛠 Технологии

- **HTML5** — семантическая разметка
- **CSS3** — стилизация (модульная структура)
- **Vanilla JavaScript** — чистый JavaScript без библиотек
- **WordPress** — будущая интеграция для администрирования контента

## 🚀 Быстрый старт

### Локальная разработка

1. Клонируйте репозиторий или откройте проект в редакторе
2. Откройте `index.html` в браузере
3. Для разработки используйте локальный сервер (например, Live Server в VS Code)

### Запуск локального сервера

```bash
# Python 3
python -m http.server 8000

# Node.js (http-server)
npx http-server

# PHP
php -S localhost:8000
```

Откройте в браузере: `http://localhost:8000`

## 🔧 Разработка

### Принципы

- **Чистый JavaScript** — без библиотек и фреймворков
- **Модульность** — код разбит на логические модули
- **BEM** — методология именования классов
- **Mobile-first** — приоритет мобильной версии
- **Совместимость с WordPress** — готовность к будущей интеграции

### Именование классов (BEM)

```html
<!-- Блок -->
<div class="card">
  <!-- Элемент -->
  <img class="card__image" src="...">
  <h3 class="card__title">...</h3>
  <!-- Модификатор -->
  <button class="card__button card__button--primary">...</button>
</div>
```

### Требования к коду

- Семантический HTML5
- Адаптивность от 320px
- Оптимизация изображений (WebP, lazy loading)
- Валидация форм на чистом JavaScript
- Доступность (a11y) — WCAG AA минимум

## 📱 Адаптивность

- **Мобильные**: 320px - 767px
- **Планшеты**: 768px - 1023px
- **Десктоп**: 1024px+

Приоритет разработки: Mobile-first

## 🔄 Интеграция с WordPress

Проект разрабатывается с учётом будущей интеграции в WordPress:

- Компоненты выделены в отдельные блоки
- Повторяющиеся части (header, footer) — в отдельные файлы
- Структура кода позволяет легко выделить компоненты для WordPress-шаблонов
- JavaScript модули готовы к подключению через `wp_enqueue_script()`
